# History Details: Supporto PWA

**Date**: 2026-06-17

## Lessons Learned & Decisions Made
1. **PWA Plugin Integration**: Integrated `vite-plugin-pwa` in `vite.config.ts` to automate service worker generation and resource precaching.
2. **Manifest Configuration**: Defined Web App manifest parameters matching the branding: name, short_name, description, theme_color (`#0f172a`), background_color (`#0f172a`), standalone display, and orientation support.
3. **Theme Color Synchronization**: Added a matching `<meta name="theme-color" content="#0f172a" />` tag to the main `index.html` to align metadata.
4. **Icon Asset Generation**: Resized the SVG favicon into 192x192 and 512x512 PNG icons using `sharp`, defining them as `any maskable` to comply with Lighthouse audits and ensure clean rendering on Android and other OS launchers.
5. **Auto-Updates**: Enabled `registerType: 'autoUpdate'` to automatically activate the service worker when assets change, ensuring a zero-friction update mechanism for users.

## Issues Encountered
- **Node Module Resolution in Scratch Script**: Executing the SVG-to-PNG conversion script from the sandbox scratch directory required setting `NODE_PATH` to point to the root `node_modules` directory to successfully resolve the `sharp` library.
